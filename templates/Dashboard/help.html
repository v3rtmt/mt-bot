Preguntas Frecuentes

Esto se debe a que la Api key o Secret key que ingresaste para realizar las ordenes no es valida o ya está expirada, genera una nueva Api key en el sitio web o aplicación de tu exchange de preferencia para poder ejecutar las ordenes con tu bot, en los bots simulados no se requiere el uso de una Api key autentica

Las API key y Secret key son utilizadas para la autenticación y autorización de acceso a la cuenta del usuario a través de la API. La API key y Secret key de los exchanges son códigos generados por el usuario a través del sitio web de se respectivo exchange, que luego se utilizan en esta aplicación para acceder a la cuenta del exchange del usuario y realizar operaciones automatizadas en su nombre. Las API key y Secret key de cada exchange son confidenciales y deben mantenerse en secreto para evitar el acceso no autorizado a la cuenta del usuario.

La mayoria de Api keys de los exchanges actuales tienen un tiempo de expiración de variando el exchange de donde generes tu Api key (comunmente siendo de 90 dias), al momento de terminarse ese tiempo de expiración no es posible hacer mas ordenes con esa Api key, si una posición esta ejecutandose al momento de que termine el tiempo de vida de la Api key, esa orden se puede cerrar con una Api key secundaria que hayas generado al saber que tu Api key actual esté a punto de vencer, de lo contrario la posición no cerrará hasta que lo hagas manualmente, es importante saber cuando creaste tus Api keys y cuando van a expirar para asi evitar perdidas de dinero.
